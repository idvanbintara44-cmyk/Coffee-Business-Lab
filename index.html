Mengapa harus mempelajari SPtLDV?

Di balik setiap secangkir kopi yang terjual, terdapat keputusan bisnis yang harus dibuat secara optimal. Sumber daya seperti bahan baku, waktu produksi, dan modal memiliki keterbatasan, sedangkan keuntungan harus dimaksimalkan. Dalam situasi inilah SPtLDV digunakan.

Apa itu SPtLDV?

Sistem Pertidaksamaan Linear Dua Variabel (SPtLDV) adalah kumpulan beberapa pertidaksamaan linear yang memiliki dua variabel (x dan y) yang mana dua atau lebih pertidaksamaan linear diterapkan secara bersamaan. Penyelesaiannya berupa suatu daerah fisibel (feasible region) pada bidang koordinat Cartesian.

Bentuk Umum
a₁x + b₁y ≤ c₁
a₂x + b₂y ≤ c₂
...
x ≥ 0, y ≥ 0

Maksimumkan Z = px + qy

Setiap pertidaksamaan merepresentasikan satu kendala (constraint), seperti modal usaha, ketersediaan bahan baku, waktu proses produksi, dll., sedangkan Z = px + qy disebut fungsi tujuan (objective function), seperti keuntungan bisnis, dan sebagainya.


Contoh Konteks Coffee Lab:
Sebuah kedai memproduksi: Es Kopi Susu (x) Americano (y) dengan kendala stok biji kopi: 15x + 20y ≤ 600.

Komponen Penting dalam Program Linear
  1. 📦 Variabel Keputusan: Jumlah produk yang akan diproduksi.
  2. 🛑 Kendala (Constraints): Keterbatasan sumber daya yang tersedia.
  3. 📈 Garis Batas: Representasi grafik dari setiap kendala.
  4. 📐 Daerah Penyelesaian: Irisan seluruh daerah yang memenuhi semua kendala.
  5. ⭐ Titik Pojok (Vertex): Titik sudut daerah penyelesaian yang menjadi kandidat solusi optimum.
  6. 🎯 Fungsi Tujuan (Objective Function): Fungsi yang akan dimaksimalkan atau diminimalkan (Contoh: Z = px + qy).
Mengapa SPtLDV Penting bagi Business Analyst?

SPtLDV merupakan fondasi Program Linear yang digunakan untuk menentukan kombinasi produksi terbaik agar keuntungan maksimum dapat diperoleh tanpa melanggar keterbatasan sumber daya.

Ilustrasi Daerah Penyelesaian
  • - Garis Batas: Garis biru (3x + y = 90) dan garis merah (x + y = 40) yang membatasi kapasitas mesin.
  • - Daerah Penyelesaian: Area transparan berwarna hijau yang dibatasi oleh titik-titik sudut.
  • - Titik Optimum: Berada di titik pojok perpotongan garis merah dan biru yaitu titik (25, 15).

Keterangan: Area berwarna menunjukkan seluruh kombinasi (x, y) yang memenuhi semua kendala.

Contoh Kasus Lengkap: Bisnis Kedai Kopi

Sebuah kedai kopi memproduksi dua jenis kopi dengan keterbatasan waktu mesin sebagai berikut:

  • 1. Mesin Roasting: Maksimal 90 jam/minggu.
  • 2. Mesin Packaging: Maksimal 40 jam/minggu.
Data Produk
  • 1. Kopi Arabika (x): Membutuhkan 3 jam roasting, 1 jam packaging, keuntungan Rp300.000/batch.
  • 2. Kopi Robusta (y): Membutuhkan 1 jam roasting, 1 jam packaging, keuntungan Rp200.000/batch.
Model Matematika
  • 1. 3x + y ≤ 90 (Kendala Mesin Roasting)
  • 2. x + y ≤ 40 (Kendala Mesin Packaging)
  • 3. x ≥ 0 ; y ≥ 0 (Kendala Non-Negatif)
Fungsi Tujuan

Z = 300.000x + 200.000y

Pembahasan Lengkap dan Langkah Penyelesaian
  1. ⚫️Membentuk Model Matematika: Mengubah narasi bisnis menjadi batasan variabel matematika sesuai poin kendala di atas.
  2. ⚫️Menggambar Grafik: Membuat garis linear untuk persamaan 3x + y = 90 dan x + y = 40 pada bidang koordinat.
  3. ⚫️Menentukan Daerah Penyelesaian: Mengarsir daerah di bawah garis yang memenuhi semua kondisi pertidaksamaan.
  4. ⚫️Menentukan Titik Pojok: Menemukan kandidat koordinat sudut luar daerah penyelesaian, yaitu: (0,0), (0,40), titik potong garis, dan (30,0).
  5. ⚫️Menentukan Titik Potong dengan Metode Eliminasi:
    3x + y = 90
    x + y = 40
    ───────── -
    2x = 50 → x = 25.
    Substitusi ke persamaan kedua: 25 + y = 40 → y = 15. Jadi, titik potong berada di (25, 15).
  6. ⚫️Menghitung Keuntungan Maksimum:
    • Titik (0, 0) → Z = 0
    • Titik (0, 40) → Z = Rp8.000.000
    • Titik (30, 0) → Z = Rp9.000.000
    • Titik (25, 15) → Z = 300.000(25) + 200.000(15) = Rp10.500.000 (Maksimum).
Metode Grafik

Setiap pertidaksamaan ax + by ≤ c digambarkan sebagai garis ax + by = c, lalu daerah yang memenuhi diarsir. Irisan semua daerah arsiran membentuk daerah fisibel.

Metode Titik Pojok

Nilai optimum fungsi tujuan linear selalu di salah satu titik pojok daerah fisibel. Hitung Z di setiap titik pojok, lalu pilih yang terbesar.

Bilangan Bulat

Dalam konteks nyata (jumlah cup tidak bisa desimal), kita perlu memastikan titik optimum berupa bilangan bulat. Batasan (c) harus dipilih agar semua perpotongan garis menghasilkan bilangan bulat.

Problem Solving Kasus Bisnis: Optimasi Kedai Kopi

Kedai kopi memproduksi Espresso dan Cappuccino. Sebagai seorang Business Analyst, ikuti langkah berpikir komputasional (Computational Thinking) berikut untuk menentukan keuntungan maksimum.

Langkah Kerja Menyelesaikan SPtLDV
Langkah 1 — Identifikasi Variabel Keputusan
  • • x: Jumlah Espresso (cup/hari)
  • • y: Jumlah Cappuccino (cup/hari)
Langkah 2 — Susun Model Matematika

Contoh:

  • 1. 15x + 20y ≤ 600
  • 2. 150x + 30y ≤ 3000
  • 3. 4x + 3y ≤ 480
  • 4. x ≥ 0
  • 5. y ≥ 0
Langkah 3 — Gambar Grafik Tiap Garis Batas

Contoh persamaan garis 15x + 20y = 600:

  • • Titik potong sumbu X: (40, 0)
  • • Titik potong sumbu Y: (0, 30)
Langkah 4 — Tentukan Daerah Penyelesaian

Daerah penyelesaian merupakan irisan seluruh daerah yang memenuhi semua kendala pertidaksamaan di atas.

Langkah 5 — Tentukan Titik-Titik Pojok

Contoh koordinat kandidat solusi:

  • 1. Titik A: (0, 0)
  • 2. Titik B: (40, 0)
  • 3. Titik C (Perpotongan): (24, 12)
  • 4. Titik D: (0, 30)
Langkah 6 — Uji Fungsi Tujuan

Contoh fungsi laba: Z = 8000x + 6000y

  • • Hasil pengujian titik optimum C: Z(24, 12) = 8000(24) + 6000(12) = 264.000

Nilai maksimum keuntungan bisnis ini diperoleh tepat pada titik optimum tersebut.

Cara Menggunakan Simulasi
  1. Baca tabel kebutuhan per cup.
  2. Isi koefisien (a dan b) setiap kendala.
  3. Gunakan slider untuk mengatur batas harian — perhatikan indikator bilangan bulat!
  4. Tuliskan prediksi x dan y optimum.
  5. Klik "Jalankan Simulasi" dan pelajari penjelasannya.
Mengapa x dan y Harus Bulat?

Jumlah cup kopi tidak bisa berupa bilangan desimal — Anda tidak bisa membuat 42.5 cup Espresso! Oleh karena itu, nilai batas (slider) harus dipilih dengan cermat agar semua titik perpotongan garis kendala menghasilkan bilangan bulat. Jika hasil perpotongan desimal, geser slider ke nilai terdekat yang menghasilkan bilangan bulat.
